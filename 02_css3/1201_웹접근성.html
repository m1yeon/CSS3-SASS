웹접근성(web accessbility)

참고사이트 : 웹접근성연구소, 한국웹접근성인증평가원, 웹와치(심사원), 네이버 널리(nully)

웹접근성이란?
장애에 구애 없이 모든 사람들이 손쉽게 정보를 공유할 수 있는 인터넷 공간 만들기
팀 버너스리 박사는 웹 콘텐츠를 제작할 때 장애에 구애없이 누구나 접근할 수 있도록 제작해야 한다라고고 함

2016.2.3 장애인 차별 금지법으로 인하여 제21조 1항에 의거하여
'누구든지 신체적/기술적 여건과 관계없이 웹사이트를 통해 원하는 서비스를 이용 할 수 있도록
접근성이 보장되는 웹사이트'라고 명시됨으로 인해 웹사이트 제작시 참고해야 함

장애인 차별 금지법으로 인해 모든 공공기관과 법인회사는 웹사이트에 웹접근성 준수 의무화가 되었습니다.

웹접근성 지침서 요약

웹접근성 지침서에는 원칙 4가지, 지침 14개, 검사항목 33가지가 있음

1. 인식의 용이성 : 사용자가 장애 유무 등에 관계없이 웹사이트에서 제공하는 모든 콘텐츠를 동등하게 인식할 수 있도록 제공하는 것을 의미함

  • 대체 텍스트 : 텍스트가 아닌 콘텐츠(이미지/오디오/콘텐츠)를 모든 사용자가 알 수 있도록 alt 속성으로 대체문자를 삽입한다
  • 멀티미디어 대체수단 : 멀티미디어 콘텐츠는 자막을 만들어서 제공한다. (영상, 사운드)
  • 적응성 : 표는 이해하기 쉽게 작성(thead, tbody, tfoot, caption 사용),
    콘텐츠는 논리적인 순서대로 작성(h1, h2, h3...)
    지시사항은 모양, 크기, 위치, 방향, 색, 소리 등에 관계없이 인식될 수 있어야 함
  • 명료성
    1)콘텐츠는 색상에 관계없이 인식될 수 있어야 한다. (그래프, 차트, 지도, 필수입력 등과 같은 항목은 색상만으로 구분하지 않도록 하고, 크기, 밑줄, 선, 기울기 등 스타일로 해결이 가능함)
    지시 사항은 모양, 크기, 위치, 방향, 색, 소리 등에 관계없이 인식될 수 있어야 함(엔티티 코드, 캐릭터 코드 사용)
    3)텍스트 콘텐츠와 배경 간의 명도 대비는 4.5 대 1 이상이어야 함(명도대비체크 도구를 사용하여 글자색, 배경색 차이를 파악하여 조정함)
    4)자동으로 소리가 재생되지 않아야 함(영상, 사운드는 자동재생이 안되도록 하고, 자동으로 하려면 반드시 muted 설정(음소거)을 넣어줘야함)
    5)이웃한 콘텐츠는 구별될 수 있어야 함

2. 운용의 용이성 : 사용자가 장애 유무 등에 관계없이 웹사이트에서 제공하는 모든 기능을 운용할 수 있도록 제공하는 것을 의미함

6.1 입력장치 접근성
1) 모든 기능은 키보드만으로도 사용할 수 이썽야 함(키보드 tab키를 눌러 포커스가 이동되는지 확인)
2) 키보드에 의한 초점은 논리적으로 이동해야 하며, 시각적으로 구별할 수 있어야 함
3) 사용자 입력 및 컨트롤은 조작 가능하도록 제공되어야 함(콘트롤할 수 있는 최소 크기를 제공하거나 여백을 활용)
4) 시간제한이 있는 콘텐츠는 응답시간을 조절할 수 있어야 함(슬라이드배너/자동슬라이드는 '플레이', '스탑'버튼 추가, 인증번호 입력양식에서는 '시간연장'메뉴를 추가한다)
5) 자동으로 변경되는 콘텐츠는 움직임을 제어할 수 있어야 함(정시나 일시정지 버튼 추가)

6.3 광과민성 발작 예방
1) 초당 3~50회 주기로 깜빡이거나 번쩍이는 콘텐츠를 제공하지 않아야 함(노약자, 장애인, 임신부 등이 놀랄 수 있기 때문에 사용하지 않음

6.4 쉬운 내비게이션
1) 콘텐츠의 반복되는 영역은 건너뛸 수 있어야 함(href속성값을 확인, skip[내비게이션 추가)
2) 페이지(titile), 프레임(iframe), 콘텐츠 블록(section, article)에는 적절한 제목을 제공해야 함
3) 링크 텍스트(a)는 용도나 목적을 이해할 숭 ㅣㅆ도록제공해야 함(a태그에 title 속성값 작성)

6.5 입력방식
1) 다중 포인터 또는 경로기반 동작을 통한 입력은 단일 포인터 입력으로도 조작할 수 있어야 함
2) 단일 포인터 입력으로 실행되는 기능은 취소할 수 있어야 함
3) 텍스트 또는 텍스트 이미지가 포함된 레이블이 있는 사용자 인터페이스 구성요소는 네임에 시각적으로 표시되는 해당 텍스트를 포함해야 함
4) 동작기반으로 작동하는 기능은 사용자 인터페이스 구성요소로 조작할 수 있고, 동작기반 기능을 비활성화할 수 있어야 함(on/off 기능의 버튼

3. 이해의 용이성(7) : 사용자가 자앵 유무 등에 관계없이 웹 사이트에서 제공하는 콘텐츠를 이해할 수 있도록 제공하는 것을 의미

7.1 가독성
1) 주로 사용하는 언어를 명시해야 함(html lang="ko")

7.2 예측 가능성
1) 사용자가 의도하지 않은 기능(새 창, 초점에 의한 맥락 변화 등)은 실행되지 않아야 함(팝업창=현재는 모달방식으로 변경됨)
2) 도움 정보가 제공되는 경우, 각 페이지에서 동일한 상대적인 순서로 접근할 수 있어야 함(푸터에 들어가는 담당자이름, 연락처, 전화번호, 이메일, 운영시간, 채팅창, ai chat 등)

7.3 입력 도움
1) 입력오류를 정정할 수 있는 방법을 제공해야 함(취소버튼, 다시쓰기)
2) 사용자 입력에는 대응하는 레이블을 제공해야 함(input 태그와 label 태그를 사용하여 연결)
3) 인증 과정은 인지 기능 테스트에만 의존해서는 안됨(비밀번호, 패턴말고 지문, 얼굴인식 기능추가)
4) 반복되는 입력 정보는 자동 입력 또는 선택 입력할 수 있어야함(쇼핑몰 온라인 구매시 자동으로 가입한 사용자 주소가 나오게하고 다른곳으로 배송시 추가 주소를 입력 할 수 있게 함)

7.4 견고성 : 사용자가 콘텐츠를 이용할 수 있도록 기술에 영향을 받지 않아야 함을 의미함, 견고성은 문법 준수, 웹 애플리케이션 접근성의 2가지 지침으로 구서오디어 있음
1) 문법준수 ) 마크업 언어의 요소는 열고 닫음, 중첩 관계 및 속성 선언에 오류가 없어야 함(html 마크업 검사를 실시하여 확인, 수정함)
2) 웹 애플리케이션 접근성 ) 콘텐츠에 포함된 웹 애플리케이션은 접근성이 있어야 함(크롬브라우저의 lighthouse 활용)
